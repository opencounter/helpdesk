Glossary of Terms

A guide to terms used throughout OpenCounter.

      Applications

    • "Application"

      An 'application' refers to any OpenCounter session initiated by a member of the public. In OpenCounter, an 'application' refers to allinformation and responses an applicant needs to provide to accomplish their goal--even if that may require securing multiple Existing Permits

    • "Applicant" or "Permit Applicant"

      An applicant or permit applicant refers to any member of the public that has initiated an OpenCounter session. Oftentimes, applicants are only using OpenCounter to learn more information--not all applicants will submit an application.

    • "Help Requests", "Notes" and the "Admin Papertrail"

      Help Requests are submitted by applicants when they are in need of assistance during an OpenCounter session. Help Requests are routed to all of the Admin Users in a particular Departmentdepending on which Page the applicant needs help with--that way, people are connected with the staff that can help, and staff doesn't have worry about routing questions to the right person. Notes are recorded by Admin Users to document communication with applicants--whether that was a response to a Help Request, or a proactive message sent to people with applicants in-progress. The Admin Papertrail is a log of all notes add by all Admin Users, which can be reviewed on an application by application basis.

    • "Forms" and "Downloads"

      OpenCounter Forms reflect your organization's Existing Permit Forms. When OpenCounter was first configured in your community, all Existing Permit Forms were collected, and broken down into their component fields. To create a form in OpenCounter, simply group together the fields that compose each existing permit form. When an application is submitted, your staff will be able to download exactly the information needed for each permit process in the "Download" section.

      Zoning

    • "Zoning District"

      The zoning districts in OpenCounter we're configured to reflect your community's zoning ordinance. They are geographic districts that determine what kind of land use is appropriate in a given location. In OpenCounter, zoning districts provide a zoning clearance for each use code.

    • Use Code

      the OpenCounter system was configured to match the existing zoning ordinance precisely. Thus, changes to the zoning districts in the OpenCounter system should probably only follow changes to the legally binding zoning ordinance itself
