Dashboard

The Dashboard provides your organization with a high-level overview of permit application activity in your community.

The Dashboard has four primary components:

  • An adjustable timeline
  • Top-level metrics during the selected timeframe
  • A map of permit applications during the selected timeframe.
  • A list of all permit applications during the selected timeframe

Taken together, these four components create a portrait of business activity in your community.

Adjust the timeframe of data reported by clicking on either end of the light blue area within the timeline at the top of the pages. Data across the rest of th page will update accordingly

Along with the number of Permit Applications initiated and submitted, OpenCounter estimates the total number of jobs created and fees incurred, based on information provided by applicants.

You can also see the status of all applications that exist within the timeframe selected, along with the contact e-mail for each Permit Applicant. To get more details about any particular application, click on the unique "ID" on the left-hand side of the table.

Applications

Here you can find details about all OpenCounter Applications in your community, along with associated Admin interactions and Forms.

The Applications home page is composed of two tables, one displaying all submitted--or finalized--applications, and another showing those still in progress. Each table displays a few high-level attributes such as the business name, the number of answers provided to date, and contact information for the Permit Applicant. Admin users can attach notes to communicate with one another, or view each application in detail.

On each individual Application page, you'll find a map showing the proposed business location, a table of fees incurred based on application responses, the information provided in response to each Field, and the sequence of Pages visited by the Applicant.

In the "Downloads" section toward the bottom of the page, you'll find links to download the "Full Application & GIS Response", as well as any Forms that Admin Users have created. These Forms are composed of select groups of Applicant answers, often corresponding to Existing Permit Forms used within Departments.

Sections

Sections are broad groups of Pages. Their primary purpose is to help applicants understand what kinds of information they’ll be asked to provide. They also serve as milestones for applicant as they proceed through the permitting process.

Some common Sections titles include:

  • A “Team” Section for Pages that contain Fields asking for information about the applicants themselves.
  • A “Location” Section for Pages that contain Fields asking for information about the prospective location of the business seeking a permit.

You can re-arrange Sections to appear in the sequence you prefer. Just drag them up and down the list using the XX symbol

You can also re-arrange the Pages within each Section---and Fields within each Page--using the same technique.

You can also upload properly structured .csv file to create a new set of Sections entirely.

Pages

Pages are just what they sound like---single web pages that Applicant’s view one at a time. You can show them to Applicants based on Visibility Rules.

Each Page is home to one or more Field---that is, one or more piece of information you are asking for from the Applicant. Pages typically host Fields that are related---and thus are titled to reflect a common topic.

Some common Page titles include:

  • Building Owner - this Page might include Fields such as:
    • Property Owner First Name
    • Property Owner Last Name
    • Property Owner Address
    • ...and so on.
  • Business Tax and Licsensing - this Page might include fields such as:
    • Business License Application Type
    • Business License Exemption Type
    • Business License Expiration
    • ...and so on.
  • Major Use Permit - this Page might include fields such as:
    • Business Parking Spaces
    • Building Stories
    • Parcel Prior Use
    • ...and on on.

You can also add an Introduction, some Additional Text, and Page Instructions to help the Applicant provide information to each Field.

Fields

Fields are the building blocks of OpenCounter. They represent a discrete piece of information you are asking from the Applicant. They are then grouped into Pages.

Fields can accept all kinds of information---but it is important for the OpenCounter to know what kind of data you are asking for. Thus, there are 9 different types of Fields.

Each Field can also have 1 - 3 special attributes.

  • Required Fields must be answered by the Applicant.
  • GIS Address Fields make calls to the GIS Server.
  • Use Code Fields help determine proper use.

For each Field, you can include a “Display lable” for Applicants as well as an internal-facing, “Admin label”. You can also include a “Field placeholder” that will show Applicant’s what type of information you are expecting. Give Applicants additional “Help info” like the kind of file formats you are looking for, if applicable.

Forms

Forms are a way to replicate your existing permitting process within OpenCounter.

Applicant responses solicited from any set of Fields can be grouped into a Form. When an Application is submitted, you’ll be able to download each Form---or group of answers---in order to easily match up with your existing permitting process and workflows.

To create or edit a Form, simply mix and match the Fields you would like to see grouped together for download.

Fees

Fees reflect your city’s Fee Schedule.

Each Fee has an associated “Base Cost”, a interval on which it recurs, and a processing time. This information let’s the Applicant know how much money and time they’ll need to make available for each Fee, and how often.

Fees can be triggered based on Requirements. When an Applicant meets certain conditions, then they alerted that they are responsible for paying the Fee. Requirements are based on the Applicant’s response to the Fields of your choice.

Use Codes

Use Codes are used to classify proposed businesses for zoning evaluation.

Use Codes help OpenCounter determine whether an Applicant is allowed to pursue their business at a given location, whether they need to meet additional requirements, or whether the proposed use is prohibited.

Each Use Code has the following attributes:

  • A numerical code
  • An Industry
  • A Sub-Industry
  • A Specific use
  • Associated keywords

Use Codes are often reflective of national business classifications standards like SIC or NAICS Codes.

Each Zoning District or special district has a judgement for each different Use Code.

Zoning Districts

Zoning Districts reflect your community’s Zoning Ordnance. Zoning determines whether businesses within each Use Code are appropriate at any given location.

These determinations are called Permissions. Within each Zoning District, each Use Code is either Permitted or Prohibited.

Each Zoning District also has the following attributes:

  • An Acronym Code
  • A Classification
  • A Name
  • A Description
  • Permissions

Departments

Departments are meant to reflect the Departments within your enterprise that are using the OpenCounter system.

Each Page is associated with one or more Department. As a result, Users that are members of Departments will recieve relevant information from Applications, and Applicants will be able to seek assistance from the appropriate staff members within your organization.

Users

A User is a member of your organization that uses OpenCounter.

Users can monitor Applications, handle Help Requests on behalf of their Department, and receive Weekly Metrics e-mails.
